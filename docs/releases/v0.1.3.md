# Problem Archive Crawler v0.1.3

Private archive-truth and operator-UX release for the PBInfo archival operator console.

Canonical tracked snapshot:

- `acceptance-20260310b`

Finalized targeted snapshot delivery:

- GitHub prerelease: `snapshot-targeted-user-20260327-0645`
- Asset: `targeted-user-20260327-0645.tar.zst`

## Highlights

- hardens the targeted PBInfo crawl and official-source harvest workflow so finalized coverage gates reflect real archive truth instead of false positives
- improves ranking/trust classification so the archive keeps the best trustworthy user source per solved language
- adds a simpler **Overview Problem Status Board** in the desktop app for:
  - solved
  - unsolved
  - complete
  - missing official source
  - missing your source
  - missing tests
- defaults the desktop app to the latest available snapshot when not manually pinned
- formalizes the rule that large finalized noncanonical snapshots should ship as release assets instead of bloating git history

## Why this release exists

The project now has two distinct responsibilities that must stay truthful:

1. **Archive correctness**
   - official sources, tests, and your own sources must be marked as missing only when they are actually missing
   - when PBInfo does not expose the content upstream, the archive must say so clearly
2. **Operator usability**
   - the desktop app must let you immediately see what is solved, unsolved, complete, or still missing without dropping straight into a heavy audit workflow

`v0.1.3` makes both of those significantly clearer.
